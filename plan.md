# План: Страница cases.html

## Цель
Создать отдельную HTML-страницу `cases.html` — профессиональную портфолио-галерею с 15 кейсами в адаптивной грид-сетке, фильтрацией по нишам, hover-эффектами и переходом из главной страницы.

---

## Шаги

### ШАГ 1 — Подготовка данных (15 кейсов)
Составить массив 15 кейсов с полями:
- `id`, `niche` (auto/beauty/med/fitness/online/food/legal)
- `company`, `city`, `metric`, `metricLabel`
- `tags`, `title`, `task` (для hover-параграфа)
- `bg` — placeholder цвет/градиент для фона ячейки

Распределение кейсов по нишам: ~2–3 кейса на нишу, 7 ниш.

---

### ШАГ 2 — Грид-сетка (оригинальная схема)
Заменить рекомендованную схему 70/30 на более кинематографичную:

**Паттерн (повторяется каждые 5 карточек):**
- Строка A: `[карточка 1 — 66%]` + `[карточка 2 — 34%]`
- Строка B: `[карточка 3 — 33%]` + `[карточка 4 — 33%]` + `[карточка 5 — 33%]`

Для 15 кейсов = 3 полных повтора паттерна. Каждый «большой» кейс (`span 2`) — более крупный с prominentным текстом.

На мобильном — все карточки в одну колонку.

---

### ШАГ 3 — HTML структура `cases.html`
- `<head>` — те же шрифты и `styles.css` + `cases.css`
- Хедер — **одинаковый** с `index.html` (копируем разметку + ссылаем на `script.js`)
- Секция фильтров: таблетки с нишами (все / авто / бьюти / мед / фитнес / онлайн / фудтех / юридические)
- Секция грида: `<section class="cases-gallery">` → `<div class="cg-grid">`
- Каждая карточка: `<article class="cg-card" data-niche="...">` со структурой:
  - `.cg-bg` — фоновый placeholder (CSS-градиент уникальный для каждой ниши)
  - `.cg-always` — нижний постоянный блок (ниша, компания, метрика)
  - `.cg-hover` — hover-блок (параграф «Задача: ...» + кнопка «Смотреть кейс →»)

---

### ШАГ 4 — CSS файл `cases.css`
- Базовые переменные и сетка грида
- `.cg-card` — position relative, overflow hidden, aspect-ratio, cursor pointer
- `.cg-bg` — placeholder градиент, transition scale на hover
- `.cg-hover` — position absolute, bottom 0, translateY(100%) → translateY(0) при hover
- Overlay затемнение на hover
- Фильтры — активное состояние с accent-цветом
- Полная адаптивность (768px → 2 колонки, 480px → 1 колонка)
- Кнопка «Смотреть кейс» в стиле существующего `.nav-link--tg`

---

### ШАГ 5 — JS файл `cases.js`
- Инициализация фильтров по нишам (show/hide карточек)
- Инициализация бургер-меню (тот же код что в `script.js`)
- Переключатель языков (упрощённая версия, только для хедера)

---

### ШАГ 6 — Обновление `index.html` и `script.js`
- Кнопка «Портфолио» в хедере `index.html` → `href="cases.html"` (убрать SPA-роутинг для этой ссылки)
- Убрать старый `#portfolio-view` блок из `index.html` (он больше не нужен)
- Обновить `script.js` — упростить `initPseudoRouting` (убрать, заменить обычной навигацией)

---

### ШАГ 7 — Плейсхолдеры фонов
Каждая ниша получает уникальный CSS-градиент вместо картинки:
- auto → тёмно-синий + холодный
- beauty → пурпурный + розовый
- med → индиго + глубокий синий
- fitness → оранжевый + янтарный
- online → зелёный акцент + тёмный
- food → тёплый тёмно-красный
- legal → серо-стальной

Большие карточки дополнительно получают крупную полупрозрачную цифру-метрику как декоративный элемент фона.
